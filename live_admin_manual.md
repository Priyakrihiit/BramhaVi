# Live Classes Platform — Administrative Manual
**Sprint 22 — Phase 9 Documentation**

## 1. Monitoring Active Meetings
*   Navigate to Django Admin panel ➔ **Live Classes**.
*   View status logs (`SCHEDULED`, `LIVE`, `COMPLETED`) for each class.

## 2. Managing Recordings and Analytics
*   Compiled stream files can be monitored under the **Recordings** table.
*   Check attendance percentages, concurrent viewers, and student engagement factors under **Meeting Analytics**.

## 3. Resolving Blocked Streams
*   If a session remains locked in `LIVE` state after broadcast termination, change the status flag to `COMPLETED` in the admin console to automatically trigger archives compilation.
